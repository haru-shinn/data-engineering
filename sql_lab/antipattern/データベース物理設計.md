# アンチパターンと解説（データベース物理設計編）

## ラウンディングエラー（丸め誤差）

人件費などの計算をする際に少数を丸めることで意図しない値が出力される。

アンチパターン: FLOATデータ型を使用する

丸めを避けることが不可
1/3 + 1/3 + 1/3 = 1.00
0.333 + 0.333 + 0.333 = 0.999

解決方法: NUMBERICデータ型を使用する

```sql
ALTER TABLE Bugs ADD COLUMN hours NUMBERIC(10, 2);
ALTER TABLE Accounts ADD COLUMN hourly_rate NUMBERIC(10, 2);
```

##  サーティワンフレーバー（31のフレーバー）

列に特定の値だけを格納する

アンチパターン: 限定する値を列定義で指定する

```sql
CREATE TABLE PersonalContacts (
  -- 他の列 . . .
  salutation VARCHAR(4)
    CHECK (salutation IN ('Mr.', 'Mrs.', 'Ms.', 'Dr.', 'Rev.'))
);
```

解決方法: 限定する値をデータで指定する

マスターテーブルを作成するイメージ

```sql
CREATE TABLE BugStatus (
  status  VARCHAR(20) PRIMARY KEY
);

INSERT INTO BugStatus (status) VALUES ('NEW'), ('IN PROGRESS'), ('FIXED');

CREATE TABLE Bugs (
  -- 他の列 . . .
  status  VARCHAR(20),
  FOREIGN KEY (status) REFERENCES BugStatus(status)
    ON UPDATE CASCADE
);
```

## ファントムファイル（幻のファイル）

画像をはじめとする大容量メディアファイルを格納する

アンチパターン: 物理ファイルの使用を必須と思い込む

解決方法: 必要に応じてBLOB型を採用する
