# OSS-DB (Silver/Gold) の学習計画と方針

設定値を「暗記」すると挫折する。
OSS-DB（特にPostgreSQL）の設定値は、**「なぜその設定が必要か」という背景（アーキテクチャ）**と紐付けるのが最短ルート。

## 学習アプローチ

- 図解で「データの流れ」を理解する
  - クエリが来てから、Shared Buffers（メモリ）に乗り、WAL（ログ）に書き込まれ、最終的にディスクへ書き出されるまでの流れを図で理解する。
- 設定値を「リソースの奪い合い」で捉える
  - work_mem を増やせばソートは早くなるが、接続数分だけメモリを消費する。
  - shared_buffers はOSとDBでメモリをどう分け合うかのバランス。
  - このように、「あっちを立てればこっちが立たず」というトレードオフの関係で覚えると、設定値の意味がスッと入る。
- 実機で SHOW all; を叩く
  - BigQuery（サーバーレス）とは違い、PostgreSQLをDocker等で立ち上げ、設定を変えて再起動する、という「手触り感」を持つのが一番の近道。
