# postgres_fdw 演習

## 演習の流れ

1. リモート側（pg_remote）にテストデータを作る
まずは接続先（ポート5436）にログインし、外部から読み込みたい本物のテーブルを作成して、サンプルデータを数件インサートします。
2. ローカル側（pg_local）に拡張機能を導入する
接続元（ポート5435）にログインし、CREATE EXTENSION postgres_fdw; を実行して、外部のPostgreSQLと通信するための機能を有効化する。
3. 外部サーバーとユーザーのマッピングを定義する
ローカル側で「どこに接続するか（ホスト名は pg_remote）」というサーバー定義を作り、次に「どのユーザーでログインするか」という認証情報を紐付ける。
4. 外部テーブル（Foreign Table）を作成してクエリを投げる
リモート側のテーブル構造を真似た「外部テーブル」をローカル側に定義する。
これが完了すると、ローカル側で SELECT を投げるだけで、自動的にリモート側のデータを引っ張ってこれるようになる。

---

コンテナの起動
```bash
docker compose up -d
```

## ステップ 1：リモート側（pg_remote）にテストデータを作る

まずはデータを持つ側のコンテナに入り、元となるテーブルとデータを作成します。

```bash
# ポート5436（リモート側）に接続
psql -h localhost -p 5436 -U remote_user -d remote_db
```

*(パスワードを求められたら remote_password と入力)*

ログインしたら、以下のSQLを実行してテーブル作成とデータ挿入を行います。

```sql
-- テスト用テーブルの作成
CREATE TABLE employees (
    name VARCHAR(50),
    department VARCHAR(50),
    salary INTEGER
);

-- サンプルデータの挿入
INSERT INTO employees (name, department, salary) VALUES
('山田 太郎', '開発部', 500000),
('鈴木 花子', '営業部', 450000),
('佐藤 次郎', '総務部', 400000);

-- 一度ログアウト
\q
```

## ステップ 2〜4：ローカル側（pg_local）での設定

ここからは、接続元となるローカル側のコンテナ（ポート5435）に入って作業します。

```bash
# ポート5435（ローカル側）に接続
psql -h localhost -p 5435 -U local_user -d local_db
```

*(パスワードを求められたら local_password と入力)*

ここからの手順は、順番がとても重要です。

1. **postgres_fdw 拡張機能の有効化:**
まずは機能自体を有効化します。

```sql
CREATE EXTENSION postgres_fdw;
CREATE SERVER remote_pg_server
FOREIGN DATA WRAPPER postgres_fdw
OPTIONS (host 'pg_remote', port '5432', dbname 'remote_db');
```

2. **ユーザーマッピングの定義:**
「ローカルの誰が、リモートの誰としてログインするか」を指定します。今回は local_user がアクセスした際、リモートの remote_user として振る舞うように設定します。

```sql
CREATE USER MAPPING FOR local_user
SERVER remote_pg_server
OPTIONS (user 'remote_user', password 'remote_password');
```

3. **外部テーブルの作成:**
リモート側にあるテーブルの「窓口」となるテーブルをローカル側に作ります。列名や型はリモート側と一致させます。

```sql
CREATE FOREIGN TABLE foreign_employees (
    name VARCHAR(50),
    department VARCHAR(50),
    salary INTEGER
)
SERVER remote_pg_server
OPTIONS (table_name 'employees');
```

## 実際に動かしてみる（演習）

設定がすべて完了しました！ローカル側（foreign_employees）に対してクエリを投げてみましょう。

### 1. データの参照 (SELECT)

```sql
SELECT * FROM foreign_employees;
```

**実行結果のイメージ:**
ローカルのDBを触っているはずなのに、先ほどリモート側に入れた山田さんたちのデータが返ってきます。

### 2. データの追加 (INSERT)

postgres_fdw は読み込みだけでなく、データの書き込みも自動で同期してくれます。

```sql
INSERT INTO foreign_employees (name, department, salary) 
VALUES ('田中 一郎', '開発部', 480000);
```

### 3. リモート側で本当に増えているか確認

一度ローカルの psql を抜けて（\q）、最初のリモート側（ポート5436）にログインし直して SELECT * FROM employees; を実行してみてください。田中一郎さんのデータがしっかり書き込まれていることが確認できます。
